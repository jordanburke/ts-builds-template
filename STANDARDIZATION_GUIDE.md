# TypeScript Project Script Standardization Guide

The canonical standardization guide now lives in **ts-builds**, the shared
toolchain — it is the single source of truth for build-script standards, tooling
configuration (tsdown, Vitest, ESLint, Prettier, TypeScript), and applying the
pattern to existing projects.

- Guide: <https://github.com/jordanburke/ts-builds/blob/main/STANDARDIZATION_GUIDE.md>
- Or invoke the **ts-builds** skill.

This template *consumes* those standards; it does not define them. Keeping one
copy in ts-builds prevents the two from drifting.
